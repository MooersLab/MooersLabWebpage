LaTeX documents on the ScholarOne Manuscript Preparation System This is a list of reminders to be reviewed before loading a manuscript written with LaTeX on the ScholarOne Manuscript Preparation System.
  • This system takes the main.tex file with the tables and figure captions deleted beforehand. This is critical. If it is not done, the latex engine used by Scholar One will try to compile the pdf with the tiff files. The latex engine cannot handle tiffs.
  • Protein Science requires bibitems be used in place of bib files.
  • The bibitems have to be in the order of citation.
  • Note note of the nonstandard format for the citations. For example, only the last abbreviation in the journal names takes a period. Failure to follow these details are a source of numerous errors that have to be fixed in galley proofs.
  • The label/ref system is broken. You have to manually number the keys to the tables and figures.
  • The system needs the psfg.bst file, which is loaded at a suppl/latex file.
  • pdflatex does not handle tiff files.
  • Protein Science requires tiff files for bitmap images.
  • The ScholarOne system will reject png files.
  • The figure captions have to be entered on-line. Take care not to copy and paste LaTeX markup in the captions. Use the html markup that is available. For example, use \verb|adsfa| for italics.
  • The tables have to be prepared in MS Word with double spacing and without internal borders.
  • The system will convert the main.tex into pdf and append the tables and tiff files.
  • The soul package allows use of the \hl{} for highlighting text. This command line-wraps the highlight. The colorbox methods fails to line-wrap the highlight. Instead, the highlight is extended beyond the right margin and right off the page.
  • The tracking of changes in a second parallel copy fails when additional errors are found after removal of the highlights.
